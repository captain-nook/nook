# 云端心流 / 船长的角落 网站架构 v2

## 1. 网站定位

**船长的角落** 是云端心流旗下的 AI 内容频道官网。

网站当前定位为：

> 一个以 AI 内容为核心的品牌官网 + 内容展示站 + 后续课程与合作转化入口。

第一版重点不是做复杂课程销售系统，而是先建立稳定的内容阵地：

- 展示品牌与频道气质
- 呈现 B 站近期视频
- 自动读取本地公众号文章库
- 提供关于我们与联系合作页面
- 保留后台文章发布能力
- 为图文、播客、课程等后续内容类型留出空间

## 3. 首页结构

首页当前采用“封面型频道首页”设计，而不是传统企业官网首页。

### 3.1 Hero 首屏

- 左侧：主标题“船长的角落”、slogan、入口按钮
- 右侧：船长头像与独立 Logo
- 视觉：浅蓝背景、柔和光斑、大留白、蓝青主色、黄色点缀

当前文案：

- 主标题：`船长的角落`
- Slogan：`在内容的海面上，找到 AI 的航向`

### 3.2 近期视频

- 视频配置在 `components/themes/HomeDefault.tsx`
- 封面使用本地文件，目录为 `public/bilibili-covers/`
- 点击视频卡片跳转到 B 站原视频页

### 3.3 公众号文章

- 首页自动读取本地公众号文章库 Markdown
- 本地目录：`D:/MMDBB_vault/20 Areas/AI_Tech/公众号文章库`
- 文章卡片链接到站内详情页 `/wechat/[YYMMDD]`
- 本地图片会复制到 `public/wechat-assets/`

### 3.4 内容形态与课程入口

首页保留以下内容形态入口：

- 视频
- 文章
- 图文
- 播客

课程区当前以正式入口形态呈现，文案为“敬请期待”，不使用过程性占位描述。

### 3.5 联系入口

首页底部提供联系合作卡片，跳转到 `/contact`。

## 4. 内容来源与维护方式

### 4.1 公众号文章

核心实现：

- `lib/local-wechat-posts.ts`
- `app/wechat/[slug]/page.tsx`

读取规则：

1. 扫描本地公众号文章库 Markdown。
2. 文件名使用 `YYMMDD_主题_公众号.md` 规则。
3. slug 取文件名前 6 位日期，例如 `260503`。
4. 标题优先读取 `publish_title`，其次读取 `title`。
5. 摘要优先读取 `publish_summary`，其次读取 `summary`。
6. 跳过通用公众号头图 `20260502142235164.jpg`。
7. 优先使用正文真实配图作为卡片封面。
8. 本地 `assets/...` 图片复制到 `public/wechat-assets/` 后再展示。

### 4.2 B 站视频

当前实现：

- 视频配置：`components/themes/HomeDefault.tsx`
- 本地封面：`public/bilibili-covers/`

维护流程：

1. 提供 B 站视频链接或 BV 号。
2. 通过 B 站接口读取标题和封面。
3. 下载封面到 `public/bilibili-covers/`。
4. 更新首页 `recentVideos` 配置。
5. 运行 `npm run build` 检查。

后续建议把视频配置抽离到 `lib/site-videos.ts`。

### 4.3 后台文章

后台文章仍保留原有数据库发布能力：

- 后台：`/admin`
- 编辑器：`/editor`
- 新建文章：`/editor?new=1`

公众号文章当前不走后台数据库，而是走本地 Markdown 自动读取。

## 6. 跨设备继续工作的关键条件

在新电脑继续工作，需要具备：

1. 能访问私有 GitHub 仓库：`https://github.com/captain-nook/yunduan_web1.0`
2. 安装 Node.js 与 npm。
3. 克隆仓库并安装依赖。
4. 如果需要显示公众号文章，需要同步本地公众号文章库：
   - `D:/MMDBB_vault/20 Areas/AI_Tech/公众号文章库`
5. 如果新电脑 vault 路径不同，需要修改：
   - `lib/local-wechat-posts.ts`
