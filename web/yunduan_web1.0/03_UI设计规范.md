# 船长的角落 UI 设计规范 v0.1

## 1. 设计目标

本网站不是传统企业官网，也不是普通博客首页，而是一个**内容型品牌门户**。

UI 目标：

- 第一眼就看出是“船长的角落”
- 风格清爽、浅色、明亮、现代
- 有内容频道感，而不是重营销感
- 适合长期更新文章、视频、图文、播客
- 为未来课程、销售、咨询留下扩展空间

## 2. 品牌定位

### 网站主品牌
- **船长的角落**

### 背后公司
- **云端心流**

### 对外呈现原则
- 首页主视觉只突出“船长的角落”
- 公司名“云端心流”放在页脚、关于页、备案区、合作页等位置
- 品牌叙事偏“内容频道 + AI观察 + 航海感”

## 3. 首页主标题与 Slogan

### 主标题
- 船长的角落

### Slogan 候选
1. **在内容的海面上，找到 AI 的航向**
2. **记录 AI 时代的观察、方法与实践**
3. **把 AI 变成更清晰的内容与更好的行动**
4. **和船长一起，驶向更有心流的 AI 世界**

### 推荐
**在内容的海面上，找到 AI 的航向**

原因：
- 有航海隐喻，和“船长”契合
- 不空泛
- 兼顾内容、AI、方向感
- 适合做首页主视觉文案

## 4. 视觉方向

### 总体气质
- 清爽
- 浅色
- 明亮
- 轻科技感
- 有秩序感
- 有一点航海/流动意象

### 关键词
- 云
- 波纹
- 航线
- 播放按钮
- 星点
- 轻风
- 海面
- 光感

### 不要的方向
- 深色沉重企业风
- 过度霓虹、赛博朋克
- 复杂花哨的动效堆叠
- 传统博客模板感
- 过重的渐变紫粉 AI 风

## 5. 色彩系统

### 主色建议
- 主蓝：`#2F80ED`
- 辅青：`#25C2A0`
- 深文字：`#132238`
- 次文字：`#4B5B73`
- 浅背景：`#F7FAFF`
- 卡片底：`#FFFFFF`
- 分割线：`#E6EEF8`

### 点缀色
- 浅天蓝：`#EAF4FF`
- 微紫点缀：`#8E9BFF`
- 柔和黄光：`#FFF3C7`

### 使用原则
- 以白底和浅蓝底为主
- 强调色只用于按钮、链接、重点标签、Hover 状态
- 渐变可用，但要克制，建议蓝青渐变为主

## 6. 字体建议

### 中文
- 优先：`PingFang SC`、`Source Han Sans SC`、`Noto Sans SC`
- 标题可加粗，正文保持轻盈

### 英文
- 只在必要处使用小写英文副标题或栏目名
- 不要让英文成为主视觉核心

### 字体策略
- 标题偏大、偏稳
- 正文行高略高，增强阅读舒适度
- 不使用过于花哨的装饰字体

## 7. 首页结构

### 7.1 Hero 首屏

构成：
- 大标题：船长的角落
- Slogan
- 一句话介绍
- 主按钮：看最新视频
- 次按钮：阅读最新文章

视觉：
- 大面积留白
- 左文右图或居中排版
- 图形元素使用云、波纹、播放键、流线
- 可有轻微漂浮感，但不喧宾夺主

### 7.2 介绍区

内容：
- 我们是谁
- 我们关注什么
- 频道核心价值
- 适合哪些人看

表现形式：
- 3 到 4 个简洁卡片
- 配图/图标风格统一

### 7.3 内容呈现区

内容类型分为：
- 视频
- 文章
- 未来扩展：图文、播客

建议采用分栏或切换标签：
- 最新视频
- 最新文章
- 未来扩展

### 7.4 视频区

视频建议：
- 以 B 站嵌入为主
- 卡片显示封面、标题、时长、简介
- 支持跳转播放，也支持内嵌播放

视频卡片风格：
- 大封面
- 圆角 16px 左右
- 轻阴影
- 悬停抬升
- 播放按钮覆盖在封面上

### 7.5 文章区

文章卡片建议：
- 标题
- 摘要
- 发布时间
- 分类标签
- 封面可选

文章风格：
- 更偏阅读感
- 不要太像资讯流
- 保持呼吸感和清晰层级

### 7.6 课程占位区

第一版先做“即将上线”的占位，不做重交易功能。

建议内容：
- 课程预告
- 工作坊预告
- 订阅通知
- 咨询入口

呈现方式：
- 轻量横幅卡片
- 小图标 + 简洁文案
- 不做强促销感

### 7.7 页脚

页脚包含：
- 云端心流公司名
- 联系方式
- 社媒入口
- 备案信息
- 版权信息

## 8. 页面类型建议

### 首页
品牌首屏 + 内容入口 + 转化入口

### 内容列表页
文章、视频、播客、图文的统一列表页

### 内容详情页
适合文章阅读，也可用于视频介绍页

### 关于页
介绍公司、团队、频道理念、合作方式

### 课程页
课程预告、报名、咨询、后续扩展入口

### 联系页
合作、商务、社群、关注方式

## 9. 卡片与组件风格

### 卡片
- 白底
- 轻阴影
- 细边框
- 大圆角
- 间距充足

### 按钮
- 主按钮：蓝色实心
- 次按钮：白底描边
- Hover 轻微变深或抬升

### 标签
- 圆角胶囊标签
- 用于分类、类型、状态

### 图标
- 统一线性图标风格
- 轻薄、圆角、简洁
- 不混用太多图标风格

## 10. 交互原则

- 轻动效，不炫技
- Hover 清晰
- 页面切换自然
- 首屏加载有轻微渐入
- 滚动出现区块时可做淡入上移
- 所有可点击元素必须可感知

## 11. 响应式原则

### 桌面端
- 首页可以更有视觉张力
- 内容区可用双栏或三栏

### 平板端
- 内容区收敛为两栏
- 保持模块间距

### 手机端
- 首屏尽量单列
- 卡片垂直堆叠
- CTA 保持明显
- 视频封面优先显示

## 12. 内容扩展预留

为未来增长预留这些能力：

- 图文内容
- 播客内容
- 课程体系
- 会员内容
- 付费咨询
- 企业服务
- 资源下载

## 13. UI 实施原则

- 先建立统一设计语言，再做页面扩展
- 先做首页和内容页，再做课程页
- 先追求整体气质统一，再追求细节特效
- 每个模块都要服务于“内容 + 品牌 + 转化”

## 14. 一句话总结

这个站点的 UI 目标不是“做得像一个模板网站”，而是做成一个**清爽、可信、适合长期内容沉淀的 AI 频道官网**。
